# Air SDK Documentation

This directory contains documentation about specific functionality of the Air SDK. The documentation is provided in the form of Jupyter notebooks that demonstrate how to use various features of the SDK with code examples.

## Building the Documentation

To build the HTML documentation:

```bash
# Install pandoc (required for notebook rendering)
brew install pandoc  # macOS
# or: apt-get install pandoc  # Ubuntu/Debian

# Build docs
make docs

# Serve locally
make docs-serve
```

## How Stubs Become Docs

The API reference pages are generated by `sphinx-autoapi`. When both a `.py` and `.pyi` file exist for the same module (e.g. `simulations.py` + `simulations.pyi`), **autoapi uses the `.pyi` stub**. This means the stubs are the source of truth for the API docs — docstrings, type annotations, and method signatures all come from the `.pyi` file.

If you need to change what shows up in the API reference, edit the stub, not the `.py`.

Modules that don't have a `.pyi` stub (like `fleets.py` and `workers.py`) fall back to whatever docstrings are in the `.py` file.

The `autoapi_ignore` list in `conf.py` controls which modules are excluded from docs entirely (e.g. `air_model`, `client`, `helpers`).
